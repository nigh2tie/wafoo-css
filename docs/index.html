Component API

拡張提案は API Spec Draft を参照

Buttons

Base: .wf-btn [.wf-btn-icon]
Variants: .wf-btn-primary .wf-btn-outline .wf-btn-subtle
Status: .wf-btn-success .wf-btn-warning .wf-btn-danger; .wf-btn-outline-*
Sizes: .wf-btn-sm .wf-btn-lg
States: .is-loading [disabled]
Tokens: --wf-primary-bg*, --wf-link-color*, --wf-font-*, --wf-btn-p* --wf-icon-*

Alerts

Base: .wf-alert
Variants: .wf-alert-success .wf-alert-warning .wf-alert-danger
Layout: .has-icon
Sizes: .wf-alert-sm .wf-alert-lg
Tokens: --wf-success*, --wf-warning*, --wf-danger*, --wf-surface-muted

Badges

Base: .wf-badge
Variants: .wf-badge-success .wf-badge-warning .wf-badge-danger
Tokens: --wf-success*, --wf-warning*, --wf-danger*

Forms

Inputs: .wf-input .wf-textarea .wf-select

Cards

Stamp

Tokens: --wf-stamp-color, --wf-stamp-weight

Steps

Design Tokens

色・余白・角丸のスケールを提供します。

/* Color (primitives) */
--wf-ink-900, 700, 500, 300, 100
--wf-accent-50 ... 900

/* Color (semantic) */
--wf-color-text, --wf-color-muted, --wf-color-accent, --wf-color-bg
--wf-link-color (links), --wf-link-color-hover
--wf-primary-bg / --wf-primary-fg
--wf-primary-bg-hover / --wf-primary-bg-active
--wf-focus (focus outline)
--wf-color-border-subtle / --wf-color-border-strong
--wf-outline-color-hover (outline button)

/* Spacing (4pxベース) */
--wf-space-1: 4px; --wf-space-2: 8px; --wf-space-3: 12px;
--wf-space-4: 16px; --wf-space-6: 24px; --wf-space-8: 32px;

/* Radius */
--wf-radius-sm: 4px; --wf-radius-md: 8px;
--wf-radius-lg: 12px; --wf-radius-full: 9999px;

/* Component tokens */
--wf-checkbox-size, --wf-radio-size: 18px;
--wf-step-bullet-size: 24px;
--wf-stamp-py / --wf-stamp-px;
--wf-shadow-subtle, --wf-overlay-light

コントラスト（WCAG 2.1）検証結果（代表値）:

  • リンク色（accent-600）vs 背景: 4.89:1（AA 達成）
  • プライマリ（白文字）vs 背景（accent-700）: 6.63:1（AA 達成）
  • コントロール境界（#8a8a8a）vs 背景: 3.33:1（非テキスト 3:1 達成）

ガイド

命名規約（Classes）

  • 接頭辞は wf- 、kebab-case（例: wf-btn , wf-card-washi ）。
  • バリアントはクラス併用（例: wf-btn wf-btn-primary ）。
  • 状態は is-* （例: is-loading , is-invalid , is-current ）。
  • ユーティリティは用途を短く（例: wf-mb-4 ）。
  • アイコンは要素内に <span class="wf-icon"> を配置。 
<a class="wf-btn wf-btn-primary is-loading">送信</a>
<div class="wf-alert wf-alert-success has-icon">...</div>

レイアウト原則

  • モバイルファースト（breakpoints: --wf-breakpoint-sm =600px, --wf-breakpoint-md =900px, --wf-breakpoint-lg =1200px, --wf-breakpoint-xl =1440px）。
  • ページ幅は .wf-container （max 960px）で整える。
  • 余白はトークン（ --wf-space-* ）とユーティリティ（ wf-mb-* ）を優先。
  • 固定高さは極力避け、コンテンツに合わせて伸縮。
  • カード（ .wf-card ）で情報ブロック、強調は .wf-card-washi 。

アクセシビリティ方針

  • コントラスト: テキストAA、非テキスト3:1 を満たす（リンク/プライマリは確認済）。
  • フォーカス: :focus-visible を削除しない。カスタム時も明確に可視化。
  • 要素の意味: ナビは <a href> 、操作は <button> を使用。
  • ラベル: for / id を関連付け、必須は .wf-required と required / aria-required を併用。
  • アイコンのみのボタンは aria-label を付与。
  • 外部リンクの target="_blank" は rel="noopener noreferrer" を付与。
  • 動き: 可能であれば prefers-reduced-motion でアニメ軽減。
  • 言語: <html lang="ja"> を維持。
