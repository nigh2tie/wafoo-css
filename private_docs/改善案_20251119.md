## 1. ユーティリティクラスの拡充

### 1.1 スペーシングの拡充

**現状の問題点:**

- スペーシング値が限定的（`wf-mt-{0,2,4,6,8,10,12}`のみ）
- Tailwindユーザーが期待する値が不足

**実装内容:**

#### 1.1.1 スペーシング値の追加

**追加する値:**

- `wf-mt-{0,1,2,3,4,6,8,10,12,16}`

**対応するCSS変数:**

```css
/* tokens.css に追加 */
--wf-space-1: 4px; /* 既存 */
--wf-space-3: 12px; /* 既存 */
--wf-space-16: 64px; /* 新規追加 */
```

**生成されるクラス:**

```css
.wf-mt-0 {
  margin-top: 0;
}
.wf-mt-1 {
  margin-top: var(--wf-space-1);
} /* 4px */
.wf-mt-2 {
  margin-top: var(--wf-space-2);
} /* 8px */
.wf-mt-3 {
  margin-top: var(--wf-space-3);
} /* 12px */
.wf-mt-4 {
  margin-top: var(--wf-space-4);
} /* 16px */
.wf-mt-6 {
  margin-top: var(--wf-space-6);
} /* 24px */
.wf-mt-8 {
  margin-top: var(--wf-space-8);
} /* 32px */
.wf-mt-10 {
  margin-top: var(--wf-space-10);
} /* 40px */
.wf-mt-12 {
  margin-top: var(--wf-space-12);
} /* 48px */
.wf-mt-16 {
  margin-top: var(--wf-space-16);
} /* 64px */
```

**実装ファイル:**

- `src/tokens.css`: `--wf-space-16` を追加
- `scripts/generate-utilities.js`: `spacing.values` を `[0, 1, 2, 3, 4, 6, 8, 10, 12, 16]` に更新

**影響範囲:**

- `margin`, `padding`, `gap` の全方向（`mt`, `mr`, `mb`, `ml`, `mx`, `my`, `pt`, `pr`, `pb`, `pl`,
  `px`, `py`, `gap`）

**優先度:** 高（Tailwind互換性の基盤）  
**実装状況:** [実装済み]（2025-11-19）

## 3. 命名規則の標準化

### 3.1 Tailwind風の命名規則の採用

**現状の問題点:**

- 命名規則が混在している
- クラス名の予測可能性が低い

**実装内容:**

#### 3.1.1 命名規則の統一

**基本方針:**

- `wf-` プレフィックスは維持（名前空間の衝突を避ける）
- Tailwind風の命名規則を採用
- クラス名の予測可能性を向上

**命名規則:**

```css
/* スペーシング */
.wf-mt-{value}     /* margin-top */
.wf-mb-{value}     /* margin-bottom */
.wf-ml-{value}     /* margin-left */
.wf-mr-{value}     /* margin-right */
.wf-mx-{value}     /* margin-x (left & right) */
.wf-my-{value}     /* margin-y (top & bottom) */
.wf-p-{value}      /* padding */
.wf-pt-{value}     /* padding-top */
.wf-pb-{value}     /* padding-bottom */
.wf-pl-{value}     /* padding-left */
.wf-pr-{value}     /* padding-right */
.wf-px-{value}     /* padding-x */
.wf-py-{value}     /* padding-y */
.wf-gap-{value}    /* gap */

/* ディスプレイ */
.wf-block          /* display: block */
.wf-inline         /* display: inline */
.wf-inline-block   /* display: inline-block */
.wf-flex           /* display: flex */
.wf-inline-flex    /* display: inline-flex */
.wf-grid           /* display: grid */
.wf-hidden         /* display: none */

/* Flex */
.wf-items-{value}      /* align-items */
.wf-justify-{value}     /* justify-content */
.wf-flex-{value}        /* flex-wrap */

/* Grid */
.wf-grid-cols-{value}  /* grid-template-columns */

/* テキスト */
.wf-text-{size}        /* font-size */
.wf-text-{color}       /* color (セマンティック) */
.wf-text-{align}       /* text-align */
.wf-font-{weight}      /* font-weight */

/* カラー */
.wf-text-{color}       /* color */
.wf-bg-{color}         /* background-color */
.wf-border-{color}     /* border-color */

/* シャドウ */
.wf-shadow-{size}      /* box-shadow */

/* レスポンシブ */
.wf-{breakpoint}-{utility}  /* sm:, md:, lg: */
```

**実装ファイル:**

- `docs/naming-conventions.md`: 新規作成（命名規則の完全なリファレンス）
- `REFERENCE.md`: 命名規則セクションを追加

**優先度:** 中（既存コードとの互換性を保ちつつ段階的に改善）  
**実装状況:** [実装済み]（2025-11-19）

### 3.2 クラス名の予測可能性向上

**現状の問題点:**

- クラス名が予測しにくい（例: `wf-w-100` vs `wf-max-w-sm`）
- Tailwindユーザーが期待する命名と異なる

**実装内容:**

#### 3.2.1 命名規則の一貫性向上

**改善例:**

```css
/* 現状（一貫性がない） */
.wf-w-100 {
  width: 100%;
}
.wf-max-w-sm {
  max-width: 480px;
}

/* 改善案（一貫性を保つ） */
.wf-w-full {
  width: 100%;
} /* Tailwind互換 */
.wf-w-screen {
  width: 100vw;
} /* Tailwind互換 */
.wf-max-w-sm {
  max-width: 480px;
} /* 維持 */
.wf-max-w-md {
  max-width: 720px;
} /* 維持 */
.wf-max-w-lg {
  max-width: 960px;
} /* 維持 */
.wf-max-w-xl {
  max-width: 1200px;
} /* 維持 */
```

**実装ファイル:**

- `scripts/generate-utilities.js`: 命名規則を統一
- `src/utilities.css`: 既存クラス名を段階的に更新（後方互換性を保つ）

**注意点:**

- 既存クラス名（`wf-w-100`）は後方互換性のため維持
- 新しいクラス名（`wf-w-full`）を追加
- ドキュメントで推奨クラス名を明記

**優先度:** 中（段階的な改善）  
**実装状況:** [実装済み]（2025-11-19）

## 注意事項

### 後方互換性

- 既存のクラス名は維持（`wf-w-100` など）
- 新しいクラス名を追加する際は、既存クラス名との競合を避ける
- ドキュメントで推奨クラス名を明記

### バンドルサイズ

- ユーティリティクラスの追加により、バンドルサイズが増加する可能性
- 将来的にcore/extras分割を検討

### テーマシステムとの統合

- カラーユーティリティは必ずテーマ変数ベースにする
- 具体的な色名（`text-red-500`など）は追加しない
- テーマ切り替えで自動的に色が変わることを保証
